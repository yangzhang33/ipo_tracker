基于当前 ipo_tracker 项目，请只实现“最佳 filing 选择器”，不要实现 parser，不要实现导出，不要实现其他 job。

请只修改或新增以下文件：
- app/parsers/filing_locator.py
- tests/test_filing_locator.py（或等价最小测试）
- README.md（如有必要）

====================
一、目标
====================

给定某个 issuer 的 filings，选择当前最适合解析的 filing。

====================
二、优先级规则
====================

优先级固定如下：

1. 424B4
2. 424B1
3. 最新的 S-1/A 或 F-1/A
4. 最新的 S-1 或 F-1

如果同类型有多个，优先 filing_date 最新的。

====================
三、实现要求
====================

实现函数：
- select_best_filing(filings: list) -> object | None

要求：
1. 输入既可以是 SQLAlchemy 对象列表，也可以是 dict 列表
2. 内部尽量写成清晰的通用逻辑
3. 对空输入返回 None
4. 代码可读，不要过度抽象

====================
四、测试要求
====================

请提供最小测试，覆盖：
1. 只有 S-1 时能选 S-1
2. 有 S-1/A 时优先 S-1/A
3. 有 424B4 时优先 424B4
4. 同类多个时选日期最新

====================
五、输出要求
====================

请输出完整代码。
不要提前实现 parsing。
